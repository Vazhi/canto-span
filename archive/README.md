# Historical recovery archive

Everything under `archive/` is retained for recovery, audit, chronology, or provenance only. It is never active project instruction and is excluded from active documentation-link validation.

- `release-history/` — obsolete root release states, pending/waiting checkpoints, prior acceptance records, and the complete historical changelog.
- `research-history/` — source-search logs, superseded research checkpoints, and restored pre-slim records.
- `documentation-pre-DOC1/` — documentation before the earlier DOC1 consolidation.
- `checkpoint-history/` — superseded checkpoint ledgers.
- `packaging-history/` — prior packaging manifests and removed-artifact ledgers.
- `tool-history/` — superseded packaging tools.
