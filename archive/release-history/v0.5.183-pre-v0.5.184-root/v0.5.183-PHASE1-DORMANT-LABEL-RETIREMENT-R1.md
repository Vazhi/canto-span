# v0.5.183 Phase 1 dormant-label retirement R1

Date: 2026-07-20

## Scope

This checkpoint performs the first retirement review required by the current Definition of Done. It changes governance records and recovery tooling only. `main.js`, `manifest.json`, and `styles.css` remain byte-identical to the frozen v0.5.183 DEMO01 candidate.

## Decision

Ten labels met all retirement conditions:

- zero accepted fixtures;
- zero verified appearances as constructions in 545 regression fixtures and 20 DEMO01 visible cases;
- no current constructor, fallback, compatibility alias, or semantic distinction depended on the label;
- the label was either a duplicate wrapper, an overbroad umbrella, or an unused future-subtype name;
- the underlying linguistic research question could be preserved under an existing canonical family without preserving the dormant active label.

## Preservation

Retirement does not discard evidence. For each label, the archive records:

- former status;
- fixture and output counts;
- remaining runtime metadata references;
- canonical family;
- mapped source identifiers;
- reason for retirement;
- the family-level research question that remains available for future work.

`BareClassifierNP` and `DemonstrativeQuantifiedClassifierNP` had plausible future research targets. The inactive names were still retired because no implemented or fixture-backed construction used them. A future candidate may create a new narrowly specified subtype after source, speaker, boundary, and implementation work begins.

## Runtime boundary

The frozen runtime still contains all ten names in `CONSTRUCTION_LABEL_REGISTRY` and embedded legitimacy metadata. `ModalResponseFragment`, `PossessiveNominalFragment`, and `ProhibitiveFragment` also remain in `CP018_FRAGMENT_CONSTRUCTION_TYPES`. These are now documented runtime residues, not active governance constructions.

They were not removed from `main.js` because doing so would alter the frozen DEMO01 candidate and its render/held-out chain. A later maintenance candidate must remove them from the active runtime registry or place necessary historical names in explicit retired-alias metadata, then rerun diagnostics and regression tests.
