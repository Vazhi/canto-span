# v0.5.183 documentation consolidation R4

## Purpose

Consolidate current authority around the newest doctrine decisions and remove stale instructions that would recreate example-specific hardcoding or request frozen second-speaker work.

## Accepted doctrine changes

- NP-internal structure is shared parser infrastructure, separate from evidence-gated construction semantics.
- The NP subsystem is its own implementation work unit.
- Complete object strings and motivating-example vocabulary are not grammatical rules.
- Unknown or low-confidence nominal fallback cannot license evidence-gated constructions.
- Unresolved `啲 A 嘅 N` attachment must remain ambiguous unless independently resolved.
- Binary nominal coordination is represented as NP coordination without inferred hidden shared material in the first scope.
- Unseen lexical items and unseen combinations are the decisive generalization test.
- Better NP coverage does not change the linguistic status of `PostverbalZoPerfectiveVP` or any other consumer.
- All second-speaker work remains frozen; the two-speaker Definition-of-Done requirement remains binding.
- Checkpoints and ledgers should be produced only for material results, not repeated process summaries.
