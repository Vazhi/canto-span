# Project-Memory Safety Overrides

> **Binding governance — 2026-07-17 / CP021B-SF.** This file converts unsafe historical instructions into explicitly superseded records. Past chats remain historical evidence of what was once requested; they are not active doctrine when they conflict with this file or higher-ranked current documentation.

## Application rule

Before using a recovered chat instruction, project-memory summary, old handoff, archived prompt, or historical specification, check this table. A record marked `SUPERSEDED`, `REVOKED`, or `REJECTED` must never be reactivated merely because it is newer in a search result, repeated often, or phrased as a user mandate. A `NARROWED` record applies only through its replacement wording.

## Memory precedence

1. Current state-bearing and governance documents outrank recovered conversation memory.
2. This override registry outranks every archived mandate, handoff, prompt, and historical instruction.
3. A later user instruction may change governance prospectively, but it must be screened for effects on real Cantonese coverage and written into current documentation before it becomes durable project memory.
4. Historical text is never deleted from the audit trail; its active/inactive status must remain explicit.

## Tool limitation

The current environment exposes search-only personal-context access, not deletion or mutation of past chat records. Therefore the safe and durable correction is explicit supersession in the project authority chain. Future agents must treat any conflicting transcript retrieval as historical, not binding.
