# Definition of Done

> **CP020R2-RA1 state — 2026-07-17:** The indefinite new-grammar freeze and replacement-only rule remain retired. Legacy Cantonese-language claims remain non-promotable until reconstructed from screened external evidence; future grammar must follow the authority-origin lifecycle. Parser `v0.5.174` is accepted after full rendered review; there is no active candidate.

## A. Research batch

A research batch is done only when:

```text
[ ] One bounded question was defined before extensive research
[ ] The question is observable and neutral; an internal observation supplied no preferred answer
[ ] Discovery terms, source classes, inclusion/exclusion criteria, and stop conditions were frozen
[ ] Every serious candidate source screened was logged, including exclusions and access failures
[ ] Language, attestation, representation, heuristic, and lexical claims were separated
[ ] Source propositions were extracted with exact locators before the project claim was written
[ ] Every language claim has explicit proposition-bearing claim–source edges
[ ] Positive, negative, ambiguous, and boundary evidence was considered
[ ] Strongest competing analysis was recorded
[ ] Design and held-out sets were kept separate
[ ] Every summary row was inspected
[ ] Every triggered/suspicious full diagnostic was inspected
[ ] At least one apparently clean full diagnostic was inspected
[ ] Mechanical results were manually adjudicated
[ ] Dispositions were recorded separately by claim layer
[ ] Runtime changes remain unauthorized until source-derived synthesis and separate implementation authorization
[ ] Frozen remediation candidates were recorded
[ ] Authoritative docs and queue were updated where justified
[ ] Checkpoint Markdown and JSON were created
[ ] Interim recovery artifacts were exported at each reached lifecycle milestone
[ ] Final recovery and rolling ZIPs were created and verified
[ ] Installable ZIP was created when distributable files changed
[ ] SHA-256 and artifact manifest were recorded
[ ] Download links were provided
```

## B. Documentation batch

A documentation batch is done only when:

- accepted baseline, governance, candidate, freeze, checkpoint, and next task agree across state-bearing files;
- authority order is explicit;
- a new model can resume without project memory;
- historical files remain historical;
- all current links resolve;
- JSON parses;
- no documentation-only batch changes runtime hashes;
- required recovery artifacts are verified and linked.

## C. Parser candidate

A parser candidate additionally requires focused, aggregate, semantic, corpus, implicated audit, syntax, fresh-extraction, and rendered semantic review as applicable. A candidate cannot snapshot and approve its own new behavior.

A grammar candidate is authorized only after the behavior originated in screened external evidence, the claim was synthesized after proposition extraction, source edges and boundaries are explicit, evaluation was frozen before implementation, and the exact runtime batch was separately authorized. CP021A authorizes no grammar candidate.

## D. Acceptance

Accepted-latest changes only after explicit acceptance evidence. Packaging and test PASS are insufficient. Research recommendations are not acceptance events.

## E. Handoff

The package must contain a self-contained `HANDOFF.md`, exact current state, authority map, master methodology, recovery protocol, active ledger, checkpoint files, next task, and verified artifacts.
