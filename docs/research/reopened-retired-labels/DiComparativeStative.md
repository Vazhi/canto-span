# DiComparativeStative

**State:** `research_pending`; no runtime authorization.  
**Historical input:** retired `ComparativeStative`.

## Narrow question

When does a Cantonese property or stative predicate followed by `啲` express comparison, and when does it instead express attenuation or scalar adjustment?

## Why reopened

The generic fallback was overbroad, but Cantonese grammatical descriptions recognize adjective/property + `啲` as a comparison strategy. Complete retirement removed the documented profile and its ambiguity.

## Existing evidence to reverify

- `SRC-LAM-2014-SURPASS-COMPARATIVE`
- `SRC-ZHENG-ZHANG-GAO-2021-HK-CANTONESE-COURSE`

## Required boundaries

Build controlled contrasts for overt comparison standards, contextual comparison, polite or imperative adjustment, lexical restrictions, and noncomparative attenuation.

## Implementation safeguard

Do not classify every stative+`啲` string as comparative. Preserve unresolved readings or return `NeedsContext` until the discourse supplies a comparison.
