# Neutral follow-up brief — 用嚟 and purposive 嚟 cluster, round two

## Purpose

Test the unresolved boundaries identified after two research rounds without revealing parser labels or expected rankings.

## Participant target

Adult native or childhood Cantonese speakers, with separate recruitment targets for:

- Hong Kong;
- Guangzhou;
- Macau;
- other Pearl River Delta Yue backgrounds.

## Goal A — resource classes

Use matched clauses with:

- physical instrument;
- money;
- bounded time;
- room/location;
- method;
- opportunity;
- animate person.

Collect naturalness plus interpretation of the initial NP:

- resource being used;
- person performing the action;
- location where the action happens;
- topic requiring context;
- another interpretation.

## Goal B — overt resource topic

Compare:

1. `Resource NP 用嚟 VP`
2. `Resource NP Agent 用嚟 VP`
3. `Agent 用 Resource NP 嚟 VP`
4. `Agent 攞 Resource NP 嚟 VP`
5. resumptive `Resource NP, Agent 用佢嚟 VP`

Include punctuation/prosody variants where possible.

## Goal C — actual versus generic use

Cross the same resource and purpose with:

1. bare `用嚟`;
2. resource-initial `用咗嚟`;
3. active `Agent 用咗 NP 嚟`;
4. active imperfective `Agent 用 NP 嚟`.

Contexts:

- conventional function;
- intended future use;
- completed actual use;
- intended purpose contradicted by actual use.

## Goal D — perfective inside purpose VP

Compare identical sentences under:

- no context;
- “What is this normally for?”;
- “What was this actually used for?”;
- contrastive intended-versus-actual context.

Forms:

1. `用嚟 VO`
2. `用嚟 V咗O`
3. `用咗嚟 VO`
4. `用咗嚟 V咗O`

## Goal E — context-recovered fragment

Test `用嚟 VP` after:

- explicit function question;
- visible referent;
- contrastive correction;
- no antecedent.

Collect antecedent choice and confidence separately from naturalness.

## Controls

- keep vocabulary common and region-neutral;
- avoid semantically impossible objects;
- do not use `空咗時間`;
- prefer `預留咗時間`, `用咗時間`, or another independently attested allocation verb;
- do not expose construction names;
- do not infer implementation decisions from results.
