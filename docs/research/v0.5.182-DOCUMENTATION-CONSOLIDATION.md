# v0.5.182 documentation consolidation

Status: **documentation-only; runtime behavior unchanged**

## Problems corrected

- Obsolete pending, waiting, evidence-limited, rejected-cycle, and prior-release files were present at the recovery root beside the accepted state.
- Mutable release facts were repeated across doctrine, evidence, workflow, validation, and root summaries.
- Historical research files contained imperative wording that could be mistaken for current authorization or next steps.
- The full changelog mixed current state with hundreds of historical checkpoint states.

## Changes

1. Rewrote `docs/current/` so each subject has one authoritative owner.
2. Made `PROJECT-STATE.md` the sole detailed owner of current runtime, counts, accepted scope, and next action.
3. Made autonomy explicit: bounded freeze/unfreeze and implementation decisions may proceed without separate user approval when governance prerequisites pass.
4. Added the coherent-batch checkpoint rule and removed any implication of rigid source, example, or time caps.
5. Reclassified `docs/research/` as evidence and decision history, not operational instruction.
6. Moved obsolete root state files into `archive/release-history/` without altering their historical text.
7. Replaced the active changelog with a concise release summary and preserved the complete original under `archive/release-history/`.
8. Corrected the current packaging builder so accepted recovery ZIPs no longer receive the obsolete `render-pending` suffix.
9. Preserved runtime bytes, accepted grammar status, evidence records, historical tests, and evaluator separation.

## Non-changes

- no parser behavior change;
- no manifest or runtime version change;
- no grammar, lexicon, Jyutping, role, scoring, or render change;
- no evaluator case opened, restored, or made reusable;
- no historical evidence deleted.
