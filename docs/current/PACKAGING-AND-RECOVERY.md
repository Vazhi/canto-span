# Packaging and recovery

## Runtime ZIP

The runtime ZIP contains exactly:

- `canto-span/main.js`
- `canto-span/manifest.json`
- `canto-span/styles.css`

It contains no research, validation, historical, speaker-review, or recovery files.

## Recovery ZIP

The recovery ZIP contains runtime source plus current doctrine, status records, tests, tools, validation, and classified history. Superseded work belongs under `archive/`.

Abandoned prospective-evaluation packets are not active inputs and must not be reused. Private or sealed evaluator outputs are excluded.

## Current checkpoint filenames

- runtime: `canto-span-plugin-v0.5.184.zip`
- recovery: `canto-span-recovery-v0.5.184-np-subsystem-r1.zip`
- full suite: `canto-span-v0.5.184-np-subsystem-r1-full-download-suite.zip`

## Checkpoint rule

Package only after a material runtime, evidence, status, retirement, or speaker-result change. Documentation-only churn does not justify repeated full-suite packaging.
